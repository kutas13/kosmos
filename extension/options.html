Kosmos başvuru doldurucu — seçenekler

API kök adresi (müşteri yükleme): örn. http://127.0.0.1:8765 — sonda / olmasın.

apiBaseUrl

Sabit alanlar her doldurmada aynı kalır. JSON dizisi: type = fill veya select, value = yazılacak değer. Alanı bulmak için name (tercih, örn. BootstrapVue name="…") veya label (etiketten parça) kullanın.

staticFields (JSON)

İsteğe bağlı: Dinamik alanlar için etiket parçaları (sayfadaki yazıyla uyuşmalı).

dynamicLabels (JSON nesnesi)

Gelişmiş: Alan başına özel strateji. Örnek: ad için CSS seçici.

dynamicFieldMap (JSON)

Sayfa 3 (adres — görsel 3): applicantCountryId (Türkiye=685), applicantCityId (İstanbul=2), street, eMail, addressClickHayir (ikamet sorusu Hayır). İsteğe bağlı: postalCode, applicantCountrySelectName (ülke select’in tam name’i).

wizardStep1Extra (JSON nesnesi)

2. adım (sihirbaz) varsayılanları üzerine yazar. Örn. passportIssueDate, birthPlace, passportNo (boş bırakırsanız her seferinde rastgele U########).

wizardStep2 (JSON nesnesi)

Sihirbaz: 4. slayt meslek (jobId), 5. slayt seyahat + parmak izi. jobId: Esnaf = "381". visaEntryTypeId: Çoklu Giriş = "58" (Tek 56, Çift 57). Vize tarihleri: visaDateAnchor (referans gün), visaBaseEntryDate / visaBaseReturnDate (o günkü giriş–çıkış, YYYY-MM-DD veya GG.AA.YYYY). Her geçen takvim günü iki tarih de +1 kayar. Sabitlemek için "visaRolling": false ve visaEntryDate, visaReturnDate verin. 7. slayt masraflar: accommodationTravelCosts (varsayılan gecimMasraflariHayir), expensePaymentCashValue (Nakit). 8. slayt KVKK: metin sona kaydırılır; kvkkLocation (ISTANBUL), kvkkLocationDate (boş = bugün).
